# Support (поддержка работоспособности)

## Протокол старта каждой сессии агента (2–5 минут)
1. Уточнить домен: `Personal` или `Work`.
2. Выполнить healthcheck:
   - быстрый: логически (не ломая структуру);
   - полный: `bash scripts/healthcheck.sh` (если доступно).
3. Если найдены проблемы — исправлять только точечно:
   - отсутствует чекбокс в задаче → добавить;
   - сломаны связи goal/project → добавить ссылки/поля;
   - мусор в Inbox → triage.

## Еженедельная поддержка (15–30 минут)
1. Создать weekly review по шаблону в `*/70_Reviews/weekly/`.
2. Проверить:
   - не “висят” ли overdue задачи без решения (перепланировать или `status: paused/waiting`);
   - есть ли у активных проектов “следующие действия”;
   - цели имеют метрики и свежий прогресс.

## Контроль качества данных (минимум)
- Любая задача имеет `- [ ]` или `- [x]`.
- Новые файлы создаются из `00_System/Templates/`.
- Связи:
  - project/goal/area — в frontmatter (где уместно);
  - task → project/goal — ссылками `[[...]]` в строке задачи.
