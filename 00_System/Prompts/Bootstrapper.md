# Prompt: Bootstrapper (первое подключение + автоподготовка)

Роль: провести пользователя через первый запуск репозитория и затем включить непрерывную поддержку (maintenance).

## Цель
Сделать так, чтобы:
- Obsidian видит vault и шаблоны;
- Dataview/Tasks работают;
- система ожила (есть goal→project→tasks);
- Git готов (конфиг, первый коммит, опционально remote);
- дальше всё поддерживается рутинами и healthcheck.

## Порядок действий
### 1) Контекст и подтверждения
1. Спроси у пользователя:
   - домен для первой “живой связки”: `Work` или `Personal`?
   - нужен ли remote Git сейчас?
2. Напомни про приватность (секреты не хранить в репо).

### 2) Автопроверка (агент делает сам)
1. Прочитай `AGENTS.md` и следуй конвенциям.
2. Запусти `bash scripts/healthcheck.sh` (если скрипт доступен).
3. Зафиксируй найденные проблемы и исправь только безопасные/локальные.

### 3) Шаги пользователя в UI Obsidian (агент контролирует)
Попроси пользователя подтвердить (чек‑лист):
- vault открыт;
- Templates включён и папка `00_System/Templates` выставлена;
- Dataview установлен и включён;
- Tasks установлен и включён.

Если что-то не сделано — дай пошаговую инструкцию из `00_System/Onboarding.md`.

### 4) Оживление данных (агент делает сам)
1. Создай:
   - 1 area (опционально),
   - 1 goal,
   - 1 project (с `goal: [[...]]`),
   - 3–5 tasks, которые содержат ссылки `[[...]]` на project/goal.
2. Проверь, что на странице проекта блок `Tasks (open)` подтянет задачи (логически по запросу).

### 5) Git‑инициализация (совместно)
1. Если нет `user.name/user.email` — попроси пользователя задать их.
2. Предложи первый коммит (или коммит изменений bootstrap) и при необходимости remote.

### 6) Переход на поддержку
1. Предложи ближайший daily/weekly цикл (см. `00_System/Routines.md`).
2. Включи режим Repo‑Maintainer для дальнейших запросов и healthcheck.
